# Architecture

Media Web Downloader is a Home Assistant add-on with a Flask UI, a persistent SQLite state store, and yt-dlp/ffmpeg workers.

## Flow

1. User submits a URL in the UI.
2. `MediaService.analyze()` validates the HTTP(S) URL, rejects credentials/custom ports, and asks yt-dlp for metadata without downloading media.
3. The UI shows available formats and presets.
4. A download/live request is queued through `JobManager`.
5. `JobManager` persists the job in SQLite, records the selected named storage, and runs work in a `ThreadPoolExecutor`.
6. yt-dlp downloads media. Progress hooks update job progress, logs, metadata, source thumbnail URL, and automatic tags.
7. ffmpeg post-processing is controlled by yt-dlp options. Local video thumbnails are generated by `FileService`.
8. Source thumbnails from yt-dlp metadata are stored separately from generated video thumbnails.
9. SQLite stores queue snapshots, full job logs, error codes, storage name, manual tags, automatic tags, thumbnails, and metadata.
10. The UI/API read jobs from `JobManager` and localized view models.

## Persistent Directories

- `/data/jobs`: SQLite database, legacy JSON migration source, queue/update state.
- `/share/youtube_downloader`: default local downloads.
- `/media/youtube_downloader`: prepared named `media` storage.
- `/media/youtube_downloader_nfs`: default named `nfs` target when NFS mode is enabled.
- `<download_dir>/.thumbnails`: generated video thumbnails, source thumbnails, and timeline thumbnails.

## Job Statuses

- `pending`: queued, not yet running.
- `downloading`: active yt-dlp/ffmpeg work or live recording.
- `waiting`: live monitor waiting for a stream to start.
- `stopping`: user requested a regular download stop.
- `completed`: output file was recorded successfully.
- `error`: failed with a user-facing message and, when known, an error code.
- `stopped`: user stopped the job.
- `interrupted`: application shutdown/restart interrupted active work.

Shutdown sets stop events, cancels retry timers, interrupts live process groups with SIGINT/SIGTERM/SIGKILL escalation, closes the executor, and stores active jobs as `interrupted`.

At startup the manager also scans local process command lines for yt-dlp processes that look like they were launched by this add-on. Matching old jobs remain `interrupted`; the process is not assumed to be healthy or attached. Its URL is used only to prevent starting a second live recording for the same source.

## SQLite And Migrations

`SQLiteStateStore` initializes SQLite with:

- `PRAGMA journal_mode=WAL`
- `PRAGMA synchronous=NORMAL`
- `PRAGMA foreign_keys=ON`

Schema migrations are additive. Existing JSON payloads remain the source of compatibility, while normalized columns are backfilled for filtering/diagnostics. Legacy `history.json` and `queue.json` are imported only when the matching SQLite tables are empty.

## Storage Model

Jobs store `storage_name` (`local`, `media`, or `nfs`). The current implementation keeps the existing single effective download directory compatible, while the named model validates and persists the selected target for future multi-directory expansion.

## Security Boundaries

- Only HTTP(S) URLs are accepted.
- URLs with embedded credentials or custom ports are rejected.
- Downloaded file access is constrained to the configured download directory.
- Thumbnail access is constrained to `.thumbnails`.
- Notifications and HA events use sanitized payloads and do not send tokens, cookies, or full source URLs.
- Home Assistant events are optional and failures do not change job status.

## Deliberately Unsupported

- DRM-protected media.
- Private/login-only media, cookies, or account-based scraping.
- Arbitrary filesystem output paths supplied by the browser.
- Secret-bearing URLs in notifications/events.
- Bypassing provider access controls.
